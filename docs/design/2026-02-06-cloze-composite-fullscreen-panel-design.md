# 完形填空与复合题全屏答题面板设计

## 背景

移动端屏幕空间有限，完形填空和复合题需要"边看题目边答题"。当前方案采用上下分屏+可拖拽面板，存在题目区和答题区互相挤压的问题。

## 设计方案：全屏弹出面板

核心思路：题目阅读和答题操作各占全屏，通过弹出面板切换。

### 1. 阅读模式（默认状态）

- 全屏显示题目内容，可自由滚动
- 完形填空：空位渲染为可点击的内联标记（未答灰色 `[第1空]`，已答绿色 `[A]`）
- 复合题：显示完整材料文本
- 底部固定操作条（56px）：
  - 左侧：进度文字（如「已答 2/5 空」）
  - 右侧：「开始作答」按钮（#00B96B）
  - 上/下一题切换按钮

### 2. 作答模式（全屏 Popup）

从底部弹出全屏面板（van-popup, position="bottom", round）：

**顶部栏**：
- 左：「查看题目」按钮（向下箭头，收起面板）
- 中：「第 X 空」或「第 X 小题」
- 右：进度圆点

**内容区**（可滚动）：
- 完形填空：当前空位的选项列表
- 复合题：子题题干 + 类型标签 + 答题区

**底部导航**：
- 上一空/题 + 进度 + 下一空/题
- 最后一项时变为「完成作答」

### 3. 交互细节

- 点击完形填空题干中的空位标记 → 弹出面板并定位到该空
- 切换空位/子题时面板不收起
- 弹出/收起动画使用 van-popup 默认过渡
