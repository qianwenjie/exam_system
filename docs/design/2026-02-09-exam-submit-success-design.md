# 考试提交成功页设计方案

> **日期**: 2026-02-09
> **状态**: 已确认，实施中
> **页面**: ExamSuccess.vue

---

## 一、设计决策

| 决策项 | 结论 |
|--------|------|
| 页面流程 | 答题页 → ExamSuccess（唯一新页面） |
| 配置切换 | 顶部可折叠工具条，浅黄色背景，原型专用 |
| 未公布状态 | 成功提示 + 公布时间卡片 |
| 成绩展示 | 同一页面渐进展示，不跳转 |
| 答案解析 | 全部直接铺开，无需任何点击交互 |
| 其他路由 | submit/result/score/review 重定向到 success |
| 状态管理 | 页面内 ref 管理配置切换，不新增 store |
| Mock 数据 | 覆盖 7 种题型各至少 1 道 |

## 二、页面结构

```
┌─────────────────────────────┐
│ 📐 原型配置工具条（可折叠）    │  ← 顶部悬浮，正式开发时移除
│  公布时机：[立即][交卷后延迟][考试结束后][结束后延迟]
│  公布内容：[仅分数][分数及对错][分数对错答案解析]
├─────────────────────────────┤
│ ✅ 提交成功区域               │  ← 成功图标 + 祝贺文案 + 提交时间
├─────────────────────────────┤
│ 📊 成绩区域（根据配置切换）    │
│  场景1: 成绩未公布 → 公布时间提示卡片
│  场景2: 仅分数 → 得分大数字 + 环形图
│  场景3: 分数+对错 → 得分 + 题目列表(✓/✗)
│  场景4: 完整信息 → 得分 + 题目完整铺开
├─────────────────────────────┤
│ 🔘 返回考试列表按钮           │  ← 底部固定
└─────────────────────────────┘
```

## 三、成绩公布配置

### 四种公布时机
1. **immediate** - 交卷后立即公布
2. **delayAfterSubmit** - 交卷后延迟N天公布（原型默认3天）
3. **afterExamEnd** - 考试时间结束后公布
4. **delayAfterExamEnd** - 考试时间结束后延迟N天公布（原型默认3天）

### 三种公布内容
1. **scoreOnly** - 仅分数
2. **scoreAndCorrect** - 分数及对错
3. **full** - 分数、对错、答案及解析

## 四、场景详细设计

### 场景1：成绩未公布
- 灰色时钟图标
- 「成绩暂未公布」标题
- 根据时机计算并显示公布日期
- 「请届时在考试列表中查看」提示

### 场景2：仅分数
- 48px 加粗主题绿色大数字显示得分
- 环形进度图（CSS实现，不引入ECharts）
- 统计行用 `--` 占位（不显示对错数量）

### 场景3：分数及对错
- 得分区域同场景2，统计行显示具体数字
- 题目列表按题型分组，每组显示小计
- ✅ 绿色满分、❌ 红色零分、⚠️ 橙色部分得分
- 题干摘要截取前20字，单行显示

### 场景4：分数+对错+答案解析
- 得分区域同场景3
- 每道题独立卡片，白色背景+圆角+浅阴影
- 选择题显示所有选项，正确答案绿色✓标记，错选红色标记
- 填空题显示每个空的正确答案和学生答案
- 简答题显示学生作答和参考答案
- 解析区域浅绿色背景 + 灯泡图标

## 五、原型配置工具条

- 默认展开，可折叠为齿轮小图标
- 浅黄色背景 + 虚线边框，明确标识为原型专用
- 按钮组使用胶囊样式，选中态主题绿色
- 切换后页面内容实时响应

## 六、路由调整

- `/exam/success/:id` → 主页面
- `/exam/submit/:id` → 重定向到 success
- `/exam/result/:id` → 重定向到 success
- `/exam/score/:id` → 重定向到 success
- `/exam/review/:id` → 重定向到 success
- ExamAnswer.vue 交卷跳转目标改为 `/exam/success/:id`
